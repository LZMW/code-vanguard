## 📋 团队概述

Code Vanguard 是一个专业的软件开发团队，专注于完成从架构设计到功能实现再到质量保障的完整开发流程。团队采用混合型协作模式，根据任务特点智能选择串行、并行或混合执行方式。

### 核心能力

- 🏗️ **系统架构设计** - 技术选型、模块设计、重构规划
- 💻 **功能实现开发** - 代码编写、算法开发、模块集成
- 🔍 **代码质量保障** - 代码审查、测试设计、性能优化
- 🔬 **技术创新调研** - 技术研究、方案探索、疑难攻关

## 🔄 协作模式详解

### 串行模式（链式传递）

```
任务A → 任务B → 任务C
  ↓       ↓       ↓
产出A → 产出B → 产出C
```

**适用场景**：
- 后续任务依赖前置产出
- 需要顺序执行的开发流程
- 阶段性项目推进

**目录结构**：
```
{项目}/.codevanguard/
└── phases/
    ├── 01_phaseA/
    │   └── INDEX.md
    ├── 02_phaseB/
    │   └── INDEX.md
    └── 03_phaseC/
        └── INDEX.md
```

---

### 并行模式（广播传递）

```
     任务A ─┐
     任务B ─┼→ 同时执行 → 汇总
     任务C ─┘
```

**适用场景**：
- 任务相互独立
- 多维度分析
- 多模块并行开发

**目录结构**：
```
{项目}/.codevanguard/
└── outputs/
    ├── memberA/
    │   └── output.md
    ├── memberB/
    │   └── output.md
    └── memberC/
        └── output.md
```

---

### 混合模式（灵活组合）

```
阶段1（串行）→ 阶段2（并行）→ 阶段3（串行）
    ↓              ↓                ↓
  准备         并行执行           汇总
```

**适用场景**：
- 复杂项目开发
- 灵活的任务编排
- 动态调整执行策略

**目录结构**：
```
{项目}/.codevanguard/
├── phases/
│   ├── 01_preparation/
│   │   └── INDEX.md
│   └── 02_summary/
│       └── INDEX.md
└── outputs/
    ├── memberA/
    ├── memberB/
    └── memberC/
```

## 📖 使用指南

### 触发协调器

直接与协调器对话即可触发团队协作：

```markdown
用户: 我需要一个完整的电商系统架构
协调器: 我将协调 Code Vanguard 团队完成此任务...
```

### 协调器工作流程

1. **需求沟通** - 使用 AskUserQuestion 确认细节
2. **模式识别** - 智能选择串行/并行/混合模式
3. **任务规划** - 生成清晰的执行计划
4. **触发专家** - 使用自然语言触发专家 agent
5. **汇总输出** - 整合所有产出交付用户

### 产出目录

所有团队产出保存在项目目录下的 `.codevanguard/` 文件夹中：

```
{项目}/.codevanguard/
├── phases/          # 串行阶段产出
├── outputs/         # 并行阶段产出
├── inbox.md         # 消息中心
└── summary.md       # 最终汇总
```
